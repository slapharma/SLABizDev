What this system does

The Anatop BD Panel automates the business development pipeline for finding and contacting pharmaceutical distribution partners for Anatop (topical diltiazem for chronic anal fissure) across 17 target countries.

Nothing is ever sent automatically. Every outreach email is created as a Gmail draft. You review it and send it yourself. The system only creates drafts.

Dashboard

The main view at sla-biz-dev.vercel.app. Shows the full pipeline state at a glance.

  • Status cards — count of leads at each stage (Qualified → Contact Found → Outreach Drafted → Sent → Replied → On Hold)
  • Pending actions — an amber box appears when something needs your attention: drafts to review, emails to mark as sent, countries not yet started
  • Countries worked — which countries have leads and how many have been sent
  • Activity feed — live log of every pipeline action (draft created, email sent, bump drafted, etc.)
Refresh the page to see the latest data — it reads directly from the Google Sheet each load.

Run Pipeline

How you kick off work on a new country or re-run a stage. Go to Run Pipeline from the dashboard or navigate to /pipeline/run.

Step 1 — Pick countries

Click individual countries to select them (they turn green with a ✓), or use the group shortcut buttons:

  • LatAm North — Mexico, Venezuela, Caribbean
  • LatAm South — Brazil, Peru, Chile, Argentina
  • Asia Pacific — Japan, South Korea, Indonesia, Laos, Thailand, Taiwan
  • Greater China — China, Taiwan
  • EMEA + Israel — South Africa, Turkey, Israel

Step 2 — Choose pipeline stage

StageWhat it does
Source companies WF2 Searches Apollo + Explorium for pharma companies with a gastroenterology focus. An AI scores each company 1–5 and writes P1/P2/P3 qualified leads to the Google Sheet.
Find contacts WF3 For each QUALIFIED lead, searches Apollo for a C-suite or Director-level contact with a verified email address. Updates the lead's Contact Name, Title, and Email columns.
Draft outreach WF4 For each CONTACT FOUND lead (P1 by default), Claude AI writes a personalised outreach email and saves it as a draft in cflack@slapharmagroup.com Gmail.
All stages (full run) Starts with sourcing. Use this for a brand-new country — stages will proceed as leads move through the pipeline.

Step 3 — Options

  • Priority filter — P1 only (score ≥ 4), P1 + P2 (score ≥ 3), or All. Draft Outreach defaults to P1 only to preserve email quality.
  • Operator — SCF (Clifton) or JS (Justin). Used to attribute activity log entries.

Step 4 — Click Run

Each selected country shows a green ✓ Started or red error after the trigger fires. n8n runs countries one at a time to stay within API rate limits — each country takes 1–3 minutes depending on how many companies are found.

After running, watch the Dashboard activity feed to see progress in real time.

Leads

A full table of every lead in the Google Sheet. Filter by status, country, or priority. Shows company name, website, contact details, and current status.

Use this to audit the pipeline, find a specific company, or check whether a contact was found.

Drafts

Shows all Gmail drafts with "Anatop" in the subject line, pulled live from cflack@slapharmagroup.com. For each draft you can:

  • Read the full email body before sending
  • Delete a draft if the quality isn't good enough — the lead stays in the Sheet at CONTACT FOUND status, so you can re-run Draft Outreach to regenerate it
Only drafts with "Anatop" in the subject are shown. Your other Gmail drafts are never touched.

Approve / Mark as Sent

After you manually send a draft from Gmail, come here to record it. This is the step that unlocks automated follow-ups.

Enter the lead ID (e.g. ANTP-MX-005) and the date you sent it. Clicking Approve does three things:

  1. Updates the lead's status to OUTREACH SENT in the Google Sheet
  2. Records the send date and last touch date
  3. Starts the follow-up cadence — n8n will automatically draft bump emails at Day +14, Day +30, and Day +60
Don't skip this step. If you send the email from Gmail but don't record it here, no follow-ups will be drafted and the lead's status will stay as OUTREACH DRAFTED.

n8n — Workflow overview

Access at sla-bd.app.n8n.cloud. You don't need n8n for day-to-day work — the webapp handles everything. But it's useful for checking run history, editing logic, or manually triggering a workflow.

#NameTriggered byWhat it does
WF1 Data Webhook All other workflows + webapp Reads and writes the Google Sheet. The central data layer for the whole system.
WF2 Country Sourcing Run Pipeline → Source companies Serper competitive web search + Apollo + Explorium company fetch + LLM qualification → appends qualified leads to Sheet.
WF3 Contact Research Run Pipeline → Find contacts Apollo people search per QUALIFIED company → updates Sheet with Contact Name, Title, Email, LinkedIn.
WF4 Outreach Drafting Run Pipeline → Draft outreach Claude Sonnet (via OpenRouter) drafts a personalised email per CONTACT FOUND lead → creates Gmail draft → updates Sheet to OUTREACH DRAFTED.
WF5 Approval Handler Approve page in webapp Updates Sheet status to OUTREACH SENT, records send date → kicks off WF6.
WF6 Follow-up Cadence WF5 (automatically) Waits 14 days → drafts bump email → waits 16 more days → drafts bump 2 → waits 30 more days → drafts final bump. Marks lead ON HOLD if no reply after all bumps.

Checking a failed run

  1. Go to sla-bd.app.n8n.cloud
  2. Click Executions in the left sidebar
  3. Find the red (failed) execution — it shows the workflow name and timestamp
  4. Click it to see which node failed and the full error message
  5. Fix the cause (see common causes below) and re-trigger from the webapp

Common causes of failure:

  • Apollo API key expired or over quota — check the Apollo dashboard
  • Google OAuth token needs re-authorising — see below
  • Explorium returned 0 results — this is not a failure, just no companies found for that country
  • OpenRouter rate limit — wait a few minutes and re-run

Re-authorising Google credentials

Google OAuth tokens typically expire after 6 months of inactivity. If Google Sheets or Gmail stops working:

  1. Go to sla-bd.app.n8n.cloud
  2. Click Credentials in the left sidebar
  3. Click Google Sheets account or Gmail account
  4. Click Reconnect and complete the OAuth flow using Sam's Google account (the account that owns the Sheet)

The Google Sheet

The sheet is the single source of truth. Every n8n workflow and the webapp reads from and writes to the same spreadsheet. Do not rename columns or tabs.

Tabs

  • Leads — one row per company. Key columns: ID, Company, Status, Priority, Country, Contact Name, Contact Email, Notes
  • Activity — timestamped log of every pipeline event
  • Countries — summary stats per country (sourced, contacted, sent counts)

Hard rules — never violated by the system

These are enforced in every n8n workflow and in every AI prompt. They cannot be overridden by changing pipeline settings.

  • 1 No auto-send. The system creates Gmail drafts only. The Gmail "send" API is never called.
  • 2 No invented facts. Every AI-written email uses only confirmed Anatop information: diltiazem formulation, Alfasigma manufacturer, Mexico track record.
  • 3 No "Anoheal". This competitor product name is banned from all AI email output.
  • 4 No Armstrong. Armstrong Pharmaceutical is a direct competitor. They are blocked from the sourcing pipeline and from outreach.
  • 5 English only. All outreach emails are written in English, regardless of the target country.
